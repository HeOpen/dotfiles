# Project Echo: Dynamic Hyprland Setup

## Overview
This repository contains the configuration files (dotfiles) for a highly optimized, dynamic desktop environment built on Wayland. The core philosophy of this "rice" is the KISS (Keep It Simple, Stupid) principle: replacing monolithic applications with focused, single-purpose CLI tools that communicate efficiently. 

The defining feature of this setup is the instant, system-wide dynamic theming. Upon selecting a wallpaper, the color palette is mathematically extracted and injected into the window manager, terminal, status bar, and system monitors in real-time.

## System Architecture

### 1. Core Environment
* **Hyprland:** A dynamic tiling Wayland compositor. It serves as the foundation of the desktop. It is configured via a Lua wrapper for advanced programmatic control over rules and variables.
* **Kitty:** A GPU-accelerated terminal emulator. It is configured to dynamically load its color palette from an external configuration file and handles image previews natively via the `icat` kitten.

### 2. User Interface & Notifications
* **Waybar:** A highly customizable Wayland status bar. It utilizes GTK3 CSS syntax and is mapped to read dynamic color variables (e.g., `@color2`, `@background`) rather than static hex codes.
* **Mako:** A lightweight notification daemon for Wayland. It receives hot-reload signals to update its borders and background colors seamlessly.

### 3. The Theming Engine
* **Pywal (`python-pywal`):** The core color extraction tool. It analyzes the selected image, extracts the 16 dominant colors, and generates configuration files and templates for the rest of the system.
* **awww:** A minimal Wayland wallpaper daemon used to draw the image to the root window with smooth transitions.
* **fzf:** A command-line fuzzy finder. It acts as the interactive frontend for the wallpaper selection process, rendering a list of files alongside a live image preview panel.
* **ImageMagick:** A background dependency utilized by Pywal for mathematical pixel analysis.

### 4. System Monitoring
* **Btop:** A C++ based resource monitor. It is configured to read a custom `.theme` template generated by Pywal, dynamically matching the UI to the current wallpaper palette.

## Example Workflow: The Glue

The system is tied together by a custom, central Bash script (`cambiar_fondo.sh`), completely replacing heavy GUI wallpaper managers. 

When invoked via a Hyprland keybind (`SUPER + SHIFT + W`), the following sequence occurs:

1. **Selection:** `fzf` opens in the terminal, displaying the `~/Pictures/wallpaper/` directory. Kitty natively renders the image previews on the right side of the screen in memory.
2. **Execution:** Upon pressing Enter, the script passes the image path to `awww` to set the background.
3. **Extraction:** `wal` processes the image in the background and generates the color cache.
4. **Injection:** * The script injects the new colors directly into Hyprland's active RAM via Inter-Process Communication (`hyprctl`).
   * It sends a `SIGUSR2` signal to Waybar, forcing it to reload its CSS.
   * It reloads `makoctl`.
5. **Persistence:** Future instances of Kitty and Btop automatically source their configurations from the newly generated Pywal cache.

This entire pipeline executes in less than a second with zero background CPU overhead once completed.
